# TypeScript Types Reference

All exported types from the starter kit.

## Config Types

Source: `src/auth/config/types.ts`

### `AuthConfig`

```typescript
interface AuthConfig {
  email: EmailConfig;
  database: DatabaseConfig;
  features: FeaturesConfig;
  session: SessionConfig;
  ui: UiConfig;
}
```

### `AuthConfigInput`

The input type accepted by `defineAuthConfig()`. Same as `AuthConfig` but with all fields optional (defaults are applied).

### `EmailConfig`

```typescript
interface EmailConfig {
  sendVerificationEmail?: SendVerificationEmailCallback;
  sendPasswordResetEmail?: SendPasswordResetEmailCallback;
}

type SendVerificationEmailCallback = (params: {
  email: string;
  url: string;
  token: string;
}) => Promise<void>;

type SendPasswordResetEmailCallback = (params: {
  email: string;
  url: string;
  token: string;
}) => Promise<void>;
```

### `DatabaseConfig`

```typescript
interface DatabaseConfig {
  adapter: "prisma" | "drizzle" | "kysely";
  url: string;
  client?: unknown;
}
```

### `SessionConfig`

```typescript
interface SessionConfig {
```

```typescript
interface FeaturesConfig {
```

```typescript
interface EmailPasswordConfig {
```


### `OAuthConfig`

```typescript
type Session = typeof auth.$Infer.Session; // Inferred from Better Auth
```

## Action Result Types

Source: `src/lib/auth/actions/types.ts`

```typescript
interface AuthSuccessResult<T = void> {
  success: true;
  message?: string;
  data?: T;
}

interface AuthErrorResult {
  success: false;
  error: {
    code: string;
    message: string;
  };
}

type AuthActionResult<T = void> = AuthSuccessResult<T> | AuthErrorResult;
```
