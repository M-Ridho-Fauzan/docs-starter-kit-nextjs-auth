# Instalasi

Jalankan Next.js 16 Auth Starter Kit di mesin Anda.

## Prasyarat

- **Node.js** 20+
- **pnpm** 10+ (`npm install -g pnpm`)
- **PostgreSQL** database (lokal atau cloud, mis. [Neon](https://neon.tech), [Supabase](https://supabase.com))
- **Git** (untuk cloning repository)

## Langkah 1: Clone Repository

```bash
git clone https://github.com/your-org/starterkit-auth-nextjs.git my-app
cd my-app
```

## Langkah 2: Instal Dependencies

```bash
pnpm install
```

Perintah ini menjalankan `postinstall` secara otomatis, yang menghasilkan Prisma client melalui `prisma generate`. Jika Anda menggunakan Drizzle atau Kysely sebagai pengganti Prisma, script `postinstall` akan melewati langkah Prisma dengan aman.

## Langkah 3: Konfigurasi Variabel Lingkungan

Salin file contoh lingkungan:

```bash
cp .env.example .env
```

Edit `.env` dengan nilai Anda. Minimal, Anda perlu:

```bash
# Required
DATABASE_URL=postgresql://user:password@host:5432/db
BETTER_AUTH_SECRET=your-secret-key-at-least-32-chars
BETTER_AUTH_URL=http://localhost:3000

# Optional — hanya jika menggunakan fitur
GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=
RESEND_API_KEY=
```

Generate `BETTER_AUTH_SECRET` yang aman:

```bash
openssl rand -base64 32
```

## Langkah 4: Sinkronisasi Database

```bash
npx prisma db push
```

Ini membuat tabel yang diperlukan (`user`, `session`, `account`, `verification`) di database Anda.

## Langkah 5: Jalankan Development Server

```bash
pnpm dev
```

Kunjungi [http://localhost:3000](http://localhost:3000). Anda akan melihat halaman login jika autentikasi dikonfigurasi dengan benar.

## Verifikasi

Setelah instalasi, konfirmasi pengaturan Anda:

1. Buka `http://localhost:3000` — akan redirect ke login atau dashboard
2. Daftarkan pengguna baru di `/register`
3. Masuk di `/login`
4. Akses dashboard di `/dashboard`

## Pemecahan Masalah

| Masalah                      | Solusi                                                            |
| ---------------------------- | ----------------------------------------------------------------- |
| `prisma generate` gagal      | Pastikan `DATABASE_URL` sudah diisi di `.env` dan PostgreSQL berjalan |
| `BETTER_AUTH_SECRET` lemah   | Generate string acak 32+ karakter                                 |
| Login selalu gagal           | Periksa `auth.config.ts` memiliki `features.emailPassword.enabled: true` |
| Halaman "not found" (404)    | Pastikan `src/proxy.ts` dikonfigurasi dengan benar sebagai middleware |
